## 完了したタスク: Inspect Code 指摘事項の修正

### 実施内容 (2026/04/15)

- **ビルド基盤の更新**: `compileSdk` および `targetSdk` を 36 に引き上げ、最新の Android 環境に対応。
- **Gradle 非推奨 API の修正**: `app/build.gradle.kts` 内の `exclude` 指定を、非推奨の multi-string
  形式から single-string 形式 (`"group:module"`) にリファクタリング。
- **スペルチェッカーのノイズ抑制**: `zulu`, `ksp`, `hilt` などのプロジェクト固有用語を
  `.idea/dictionaries/project.xml` に登録し、偽陽性の警告を排除。
- **XML 検証**: 指摘されていた空タグの問題が、プロジェクトのソースコードではなくインスペクションレポート自体に含まれる
  XML に起因するものであることを確認。ソース側には問題がないことを担保。
- **ドキュメント更新**: `docs/features/09_INSPECT_CODE_FIX.md` に修正結果と、AGP
  内部に起因する修正不可な警告（保留事項）を記録。

### 反省・留意事項

- **AGP 内部警告の扱い**: `lint-gradle` や `aapt2` に関する警告は Android Gradle Plugin (8.9.1)
  の内部実装に依存しており、プロジェクト側では修正不能であることを突き止めた。無理に修正を試みず、適切に「保留（将来の
  AGP 更新待ち）」としてドキュメント化した点は良い判断だった。
- **テストによる担保**: 大規模な依存関係や SDK の更新を行った後は、必ず `testDevDebugUnitTest`
  などのテストタスクを実行し、デグレがないことを確認する手順を徹底した。
