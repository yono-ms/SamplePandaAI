# 詳細設計: 国際化 (i18n) の対応

## 1. ゴール

アプリ内のすべてのユーザー向け文字列を `strings.xml` から取得するようにし、多言語対応の基盤を構築する。特に文字数変化の大きい言語や
RTL 言語を含めることで、堅牢な UI 実装を担保する。

## 2. 影響範囲

- **UI Layer**: 既存の Compose 画面（`UserNameInputScreen`, `RepoListScreen`,
  `UserNameHistoryScreen`, `LicenseScreen`）および共通コンポーネント。
- **Resource**:
    - `res/values/strings.xml` (日本語 - デフォルト)
    - `res/values-en/strings.xml` (英語)
    - `res/values-zh-rCN/strings.xml` (中国語 簡体字)
    - `res/values-de/strings.xml` (ドイツ語 - テキスト拡張の検証用)
    - `res/values-ar/strings.xml` (アラビア語 - RTL レイアウトの検証用)
- **Test**: `androidTest` 内の文字列照合を行っているテストコード。

## 3. 現状調査結果

### 3.1 `strings.xml` の定義状況

主要な文字列は既に `strings.xml` に定義されているが、一部の動的な文字列やテストコード、および以下の箇所でハードコードが確認された。

### 3.2 ハードコード箇所の特定

- `RepoListComponents.kt`: "Retry" ボタンのテキスト。
- `RepoListScreen.kt`: "GitHub Repositories: $username" というフォーマット文字列。
- 各 `androidTest`: `onNodeWithText("...")` で日本語や英語が直接指定されている。

## 4. 実装方針

### 4.1 リソース管理

- **対応言語の選定理由**:
    - **日本語 (`ja`)**: 開発メイン言語。
    - **英語 (`en`)**: グローバル標準。
    - **中国語 (簡体字 `zh-rCN`)**: 全角文字の密度確認。
    - **ドイツ語 (`de`)**: **テキスト拡張 (Text Expansion)** の代表例。英語より 20-30%
      長くなる傾向があり、ボタン等のはみ出し確認に最適。
    - **アラビア語 (`ar`)**: **RTL (Right-to-Left)** の代表例。UI の左右反転が正しく行われるかの確認用。
- **プレースホルダー**: 動的な値（ユーザー名、エラーメッセージ等）を含む文字列は `%s` や `%d`
  を使用するように統一する。

### 4.2 修正が必要な文字列キー (追加・更新分)

| キー名               | 日本語                | 英語                        | 中国語             | ドイツ語                     | アラビア語                 |
|:------------------|:-------------------|:--------------------------|:----------------|:-------------------------|:----------------------|
| `repo_list_title` | `GitHub リポジトリ: %s` | `GitHub Repositories: %s` | `GitHub 仓库: %s` | `GitHub-Repositorys: %s` | `مستودعات GitHub: %s` |
| `error_prefix`    | `エラー: %s`          | `Error: %s`               | `错误: %s`        | `Fehler: %s`             | `خطأ: %s`             |
| `retry_button`    | `再試行`              | `Retry`                   | `重试`            | `Wiederholen`            | `إعادة المحاولة`      |
| `star_count`      | `★ %d`             | `★ %d`                    | `★ %d`          | `★ %d`                   | `★ %d`                |

### 4.3 UIコードの修正

- 全ての `Text(text = "...")` を `stringResource(id = R.string.xxx)` に置き換える。
- フォーマットが必要な場合は `stringResource(id = R.string.xxx, arg1)` を使用する。

### 4.4 テストコードの対応（二段構えの検証）

「リソースIDとの紐付け」と「実際の表示内容（翻訳・結合）」の両方を担保する。

1. **リソースIDによる検証（不変性）**:
    - コード内でハードコードが排除され、`R.string.xxx` が参照されていることを確認する。
2. **英語ロケール固定でのリテラル検証（正当性）**:
    - `androidTest` 実行時に `Locale.ENGLISH` を強制設定し、期待する英語文字列が表示されているか
      `onNodeWithText` で検証する。

### 4.5 長文・RTL への対応（レイアウト崩れ対策）

- **Flexible Layout**: 固定幅を避け、`fillMaxWidth` や `weight` を活用。
- **Text Overflow**: 1行必須の箇所では `Ellipsis` と `maxLines = 1` を指定。
- **Compose Preview での全言語検証**:
    - **対応言語すべて (ja, en, zh, de, ar)** を網羅した Multipreview を作成する。
    - 特にドイツ語での文字溢れと、アラビア語での RTL ミラーリングを重点的に確認する。
    - フォントスケール 1.5倍の設定も併せて確認する。

## 5. テスト観点

- [ ] 全ての対応言語で、文字列が正しく翻訳・表示されること。
- [ ] ドイツ語等の長文言語で、ボタンやラベルがはみ出さず適切に処理されていること。
- [ ] アラビア語でレイアウトが左右反転し、意図通り表示されること。
- [ ] **全言語プレビューの実施**: 全画面の Preview で全ロケールを一括確認すること。
