# 国際化 (i18n) 対応

## 1. ゴール (Goal)

- アプリ内のすべての文言をリソースファイル (`strings.xml`) に移行し、ハードコードを排除する。
- 複数言語（日本語、英語、中国語、ドイツ語、アラビア語）への対応準備を整える。
- ロケール切り替え時に UI が適切に更新されることを確認する。

## 2. 影響範囲 (Scope)

- `app/src/main/res/values/strings.xml` (デフォルト: 英語)
- `app/src/main/res/values-ja/strings.xml` (日本語)
- `app/src/main/res/values-zh/strings.xml` (中国語)
- `app/src/main/res/values-de/strings.xml` (ドイツ語)
- `app/src/main/res/values-ar/strings.xml` (アラビア語)
- `app/src/main/java/com/example/samplepandaai/ui/` 配下の Compose UI コンポーネント

## 3. 実装内容

### 文言の定義

以下のキーで文言を定義する。

| キー                      | 英語 (Default)              | 日本語             | 備考 |
|:------------------------|:--------------------------|:----------------|:---|
| `app_name`              | SamplePandaAI             | SamplePandaAI   |    |
| `input_user_name_hint`  | Enter GitHub User Name    | GitHub ユーザー名を入力 |    |
| `search_button`         | Search                    | 検索              |    |
| `history_title`         | Search History            | 検索履歴            |    |
| `error_empty_user_name` | User name cannot be empty | ユーザー名を入力してください  |    |
| `error_user_not_found`  | User not found            | ユーザーが見つかりません    |    |
| `license_title`         | OSS Licenses              | オープンソースライセンス    |    |
| `retry_button`          | Retry                     | 再試行             |    |

### UI での利用方法

```kotlin
Text(text = stringResource(id = R.string.input_user_name_hint))
```

### 右から左 (RTL) への対応

アラビア語対応のため、レイアウトには `Modifier.fillMaxWidth()` 等を適切に使用し、`Start`/`End` を用いた制約（
`Left`/`Right` ではなく）を徹底する。

## 4. テスト観点

- [ ] `values-ja`, `values-en` 等の各リソースファイルが正しく読み込まれるか。
- [ ] 端末の設定言語を変更した際、アプリの表示言語が即座に（または再起動後に）切り替わるか。
- [ ] アラビア語設定時にレイアウトが左右反転（RTL）し、崩れがないか。
- [ ] 未定義の言語（例: フランス語）の場合、デフォルトの英語が表示されるか。

## 5. 移行手順

1. `strings.xml` に文言を追加。
2. 既存の Compose ファイル内の String リテラルを `stringResource()` に置換。
3. Preview で各ロケールの表示を確認。
4. インストゥルメントテストでロケール切り替えをエミュレートして検証。
