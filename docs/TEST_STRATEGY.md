# テスト戦略 (Test Strategy)

このドキュメントでは、SamplePandaAI プロジェクトにおける品質担保のためのテスト観点と、現在の実装状況を定義します。

## 1. テストの方針

- **自動テストの重視**: 手動テストを最小限にし、CI/CD で実行可能な自動テストを主体とする。
- **レイヤー別検証**: クリーンアーキテクチャの各レイヤーにおいて、適切な粒度のテストを実施する。
- **実データに近い検証**: Data層では `MockEngine` を用い、ネットワーク通信に近い形式でパースやシリアライズを検証する。
- **信頼性の担保**: AI エージェントが生成したコードは、必ずこれらのテストをパスすることを条件とする。

## 3. 今後の課題・不足しているテスト

1. **エラーハンドリングの網羅 (D-5)**:
    - HTTP 404 や 500 などのエラーレスポンスを受け取った際の `GitHubRepositoryImpl`
      の振る舞いを確認するテストが不足している。
2. **エッジケースの検証**:
    - 通信タイムアウト時の挙動。
    - リポジトリ名が極端に長い場合や、画像 URL が無効な場合の UI 崩れ（スクリーンショットテストの導入検討）。
3. **カバレッジの可視化**:
    - 現在は主要なパスを網羅しているが、Jacoco 等を用いたカバレッジ測定は未導入。
